# Coding Standards

## TypeScript
- Strict mode is enforced — no implicit `any`, no type assertions without justification
- Verify type correctness with `npx tsc --noEmit` after changes
- Prefer explicit return types on public functions and component props interfaces

## Styling
- **Panda CSS only** — import utilities from `styled-system/css` or `styled-system/patterns`
- No CSS modules, no styled-components, no inline `style` props
- Scoped `<style>` blocks in `.astro` files are acceptable for layout-level styles
- Never edit anything under `styled-system/` — it is auto-generated by `npm run prepare`

## Formatting
- Prettier handles all formatting — run `npm run format` to auto-fix
- Config: single quotes, semicolons, trailing commas, 100-char print width, 2-space indent
- Astro files use `prettier-plugin-astro`; always run format before committing

## Linting
- ESLint with `typescript-eslint` and `eslint-plugin-react-hooks`
- Run `npm run lint` to check both ESLint and Prettier compliance
- Fix all lint errors before committing; warnings should be minimized

## Component Rules
- Use `.astro` components for static/layout elements with no interactivity
- Use React (`.tsx`) components for anything requiring client-side state or effects
- Keep components focused — one responsibility per file
- Do not add new packages without clear justification
